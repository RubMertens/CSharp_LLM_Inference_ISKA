Tokenization

Models don't read text — they read numbers.

Tokenizer Example
"Hello world"  →  [15496, 995]
"Hello"        →  [15496]
" world"       →  [995]

How BPE Works

Start with individual characters. Find the most common pair. Merge it into one token. Repeat thousands of times.

Like text compression, but for neural networks.

Why It Matters

Token count affects speed, cost, and context window. Fewer tokens = faster inference.

~4 characters per token on average for English.
